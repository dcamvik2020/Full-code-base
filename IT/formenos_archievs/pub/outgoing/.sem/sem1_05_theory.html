  • Алгоритм Евклида. Выкладки док-ва неточной оценки сложности второй версии алгоритма (которая через деление): Пусть a делим на b, тогда выполняется соотношение a = bq + r (1), где для остатка всегда 0 ≤ r < b (2). Поскольку a делится нацело на НОД, b тоже, значит и r придётся разделиться. Поэтому НОД(a, b) = НОД(r, b). Подставим (2) в (1): a = bq + r > rq + r = r(q + 1) /* частное явно q ≥ 1 */ ≥ 2r => r < a/2, rb < ab/2 Отсюда видно, что на каждом шагу произведение чисел ab уменьшается больше, чем вдвое. Поэтому за n шагов, сведя его к 1, получим, что ab > 2n, n < log2(ab) = log2(a) + log2(b). Ответ O(log(a) + log(b)). А по Т.Ламе O(log(min(a,b))).

  • Решето Эратосфена. Применяется, когда надо найти сразу несколько простых чисел, причём их значение ограничено (!) числом N. Однако, оно нерационально для тех случаев, когда нам надо всего лишь определить, простое ли данное число, или для разложения данного числа на множители.
