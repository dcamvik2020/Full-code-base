• В Eclipse делаем так: выделяем текущий проект teststat, затем в меню Project -> Properties => в окошке слева C/C++ Build -> Settings => в окошке справа Tool Settings -> GCC/MinGW C Linker -> в поле Libraries (-l) добавить stat, а в поле Library search path (-L) - путь к собранной библиотеке (можно относительный, например, "${workspace_loc:/stat/Debug}"). Для конфигурации Release заменить соответственно на Release. Можно ещё в Project References выставить зависимость проекта teststat от проекта stat, чтобы при компиляции первого компилировался и второй.
  • В Code::Blocks: Project -> Build options на вкладке Linker settings в окошке Link libraries добавить нашу статическую библиотеку, вместе с относительным путём до неё (..\stat\bin\Debug\libstat.a).
  • В QtCreator: По проекту ПКМ -> "Добавить библиотеку" и далее выбрать пункт "Внешняя", затем настраивать диалог. Но проще вручную прописать в файл проект.pro строчку LIBS += /путь/до/libбиблиотеки.a а если она лежит в папке с проектом, то LIBS += $$PWD/libбиблиотека.a либо в формате с ключами -L и -l .
  • В M$ Visual C++: заходим в свойства проекта Project -> teststat Properties => в диалоге раскрываем пункт Configuration Properties -> Linker -> Input и в поле Additional Dependencies прописываем stat.lib . Если путь до неё не указали, то надо ещё в Configuration Properties -> Linker -> General в поле Additional Library Directories добавить путь ..\Debug Ну или ..\Release, если выбрана соответствующая конфигурация. А можно поступить проще: вместо всего этого поставить зависимость проекта teststat от stat: выделить проект teststat, пункт меню Project -> Project Dependencies и поставить нужную галочку. Тогда система сама проставит нужные ссылки.
