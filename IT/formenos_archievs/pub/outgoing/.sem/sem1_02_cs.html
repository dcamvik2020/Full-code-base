Придерживайтесь, пожалуйста, следующих правил хорошего стиля программирования. Если Ваше решение будет плохо оформлено, то первым же (нет, нулевым) заданием на сдаче для Вас будет "отформатировать текст программы в соответствии с правилами Code Style":
  1. Отступы, скобочки { }, пробелы - это то, что проверяется Style Checker'ом автоматически. Правила вы можете почитать здесь. (Есть такой замечательный язык - Python. Там нет скобочек { }, там тела операторов выделяются отступами. Поставил неправильный отступ - программа не будет работать, или будет, но неправильно.)
  2. Не должно быть более одного пробела подряд, если это не отступы и не комментарии.
  3. Комментарии. Много не бывает. Особенно надо комментировать нетривиальные преобразования, в т.ч. адресную арифметику, особенно, если используются Сишные сокращения ("x" вместо "x != 0", "x ^= y" вместо "x = x ^ y"). Если используются глобальные переменные, каждую прокомментировать, для чего они, какие значения принимают (и что эти значения означают), при каких условиях изменяются. Т.е. полноценно документировать. До заголовков функций также желательно кратко описать, что за функция, что значит каждый параметр, и что значит каждое возвращаемое значение.
