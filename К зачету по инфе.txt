.. Реальный режим:
	-сегменты по 64 КБ (совсем немного)
	-сегментная адресация памяти(адрес = 2 числа по 16 бит: нач. сегмента, сдвинутого на 4 бита(2^4=16), и смещения ячейки от нач. сегм.)
		физич/лин адрес = сегментная_часть * 16 + смещение;
		если в коде инструкции не указана сегментная часть адреса, то  процессор ищет ее из одного из сегментных регистров(зависит от кода инструкции)
		адреса 0400h:0001h и 0000h:4001h ссылаются на одну ячейку памяти("h" = "0x", т.е. 16чная с.с.)
		min adress = 0000h:0000h = 0;      max adress = FFFFh:FFFFh = FFFF0h + FFFFh = 10FFEFh = 1 114 095 Б = 1 МБ + 64 КБ - 17 Б (+1 первый байт еще, если число адресов)
		при переполнении адресов старший бит игнорируется - зацикливается, т.е. можно адресовать ...(см на пред. строке) памяти
		однако, в процессорах 8086/8088 адресных линий на шине адресов всего 20, поэтому адресовать можно 2^20 Б = 1 МиБ памяти
	-любому процессу досутпна вся память компьютера
	-нет никакой защиты процессов друг от друга(названия раньше режим не имел, появился после созд-я "защищенного", они могут перекрываться
	      нужного для защиты процессов друг от друга, т.е. позволить процессам не лезть в память дургого процесса)
	-для процессоров <= 80286 (до 80386) защищ. реж. != защищ. реж., т.к. эти процессоры не поддерживали страничную организацию памяти
.. Защищенный режим:
	-используются сегменты по 4 ГБ в соответствии с разрядностью адресной шины
	-реализуется механиз виртуальной памяти -> распределение физической памяти, поддержка работы программ
	-линейный адрес не равен физическому, он виртуальный, для преобразования в физический используется страничное преобразование адреса
	-каждый сегмент характеризуется дескриптором(64 разряда), системную структуру данных


.. Объявление - говорим, что вот это просто может встретиться дальше (никакой паямти не выделится) (хвалимся, что у нас на руках есть 100$). ---- int a;
   Инициализация - присваивание сразу при объявлении (память выделяется) (заяляем, что у нас есть 100$). ---- int a = 5;
   Описание = определение (присваивание) (память выделяется) (тратим все 100 долларов). ----a = 5;

.. Динамические библиотеки(.dll):
	-это файл с машинным кодом, загружающийся в память процесса либо при создании, либо по запросу(динамически)
	-можно указать, какая функция нужна из библиотеки, причем ни машинный код, ни исходный текст функции не попадут в состав программы
	-может использоваться одна копия несколькими процессами
	-исправить ошибки просто(достаточно заменить файл библиотеки, перезаупустить программы)
	-однако, при изменении файла библиотеки программы могут перестать работать
	-разные программы могут ожидать разные версии библиотеки

.. Статические библиотеки(.lib):
	-файл с исходным кодом или объектный файл, предназначенный для подстановки в момент компановки
	-преобразуются компилятором в объектные файлы, чтобы слинковаться с объектными фалйами нашей программы во время создания исполняемого файла
	-если библиотека уже в виде объектника была, то она уже готова к компановке
	-все необходимые функции вставляются в один исполняемый файл
	-увеличение веса исполняемого файла на диске или в памяти
	-если появилась ошибка в библиотеке, то необходима перекомпиляция всех программ
